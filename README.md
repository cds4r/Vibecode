# VibeVPN — продажа VPN-подписок (3x-ui) + Telegram-бот

Готовый сервис для продажи подписок на VPN: **сайт** с тарифами и оформлением, **REST API** и **Telegram-бот**. Клиенты создаются автоматически в панели **3x-ui** (Xray / VLESS + Reality), после оплаты пользователю мгновенно выдаётся ключ и QR-код.

Работает «из коробки» в **демо-режиме** (без реальной панели и платежей), а в боевом — подключается к вашей панели 3x-ui и платёжной системе через переменные окружения.

## Возможности

- 🛒 Сайт-лендинг с тарифами, оформлением заказа и просмотром подписки (ключ, QR, ссылка-подписка, остаток трафика).
- 🤖 Telegram-бот: покупка, «Мои подписки», выдача ключа с QR, статус трафика.
- 🔌 Интеграция с 3x-ui: логин по сессии, создание клиента в инбаунде, чтение трафика, генерация `vless://` ссылки.
- 💳 Оплата: звёзды Telegram (`XTR`, без провайдера), платёжный провайдер (`BOT_PROVIDER_TOKEN`) или демо-режим.
- 🧪 MOCK-режим: если панель/оплата не настроены — выдаются демо-ключи, весь флоу можно протестировать.
- 🗂 Хранение заказов/подписок/пользователей в простом JSON-файле (без внешней БД).

## Технологии

Node.js (ES-модули), Express, Telegraf, qrcode. Внешних БД и нативных модулей нет.

## Быстрый старт

```bash
npm install
cp .env.example .env   # при необходимости отредактируйте
npm start
```

Откройте `http://localhost:3000`. Без заполнения `.env` сервис поднимется в демо-режиме (mock-панель, демо-оплата, бот выключен).

## Настройка боевого режима

Отредактируйте `.env` (см. `.env.example`):

### 1. Панель 3x-ui
```
PANEL_URL=https://ваш-домен:2053
PANEL_USERNAME=admin
PANEL_PASSWORD=пароль
PANEL_INBOUND_ID=1          # ID инбаунда, куда добавлять клиентов
NODE_HOST=vpn.вашдомен.ру   # хост в ссылках (если отличается от панели)
SUB_BASE_URL=https://ваш-домен:2096/sub   # если включён сервис подписок
```
Клиенты добавляются через `POST /panel/api/inbounds/addClient`, трафик читается через `getClientTraffics/{email}`.

### 2. Telegram-бот
```
BOT_TOKEN=токен_от_BotFather
BOT_PAYMENT=stars           # stars | provider | mock
BOT_PROVIDER_TOKEN=         # нужен только при BOT_PAYMENT=provider
BOT_CURRENCY=RUB
ADMIN_TELEGRAM_IDS=12345678 # кому слать уведомления о покупках
```
- `stars` — оплата звёздами Telegram, ничего дополнительно подключать не нужно.
- `provider` — оплата картой через провайдера (получите токен в @BotFather → Payments).
- `mock` — демо-кнопка подтверждения (требует `ALLOW_MOCK_PAY=true`, только для тестов).

### 3. Безопасность
- В бою поставьте `ALLOW_MOCK_PAY=false`, чтобы отключить демо-оплату на сайте/в боте.
- Задайте свой `ADMIN_KEY` — он защищает эндпоинты `/api/admin/*`.

## Тарифы

Редактируются в `src/plans.js` (название, срок, цена в ₽ и звёздах, лимит трафика/устройств, описание).

## Личный кабинет и админка

- **Личный кабинет** (кнопка «Кабинет» на сайте): регистрация/вход по email + паролю, все подписки в аккаунте (доступны с любого устройства), продление, копирование ключей. Покупки залогиненного пользователя автоматически привязываются к аккаунту; анонимные подписки этого устройства подхватываются при входе.
- **Админ-панель**: страница `/admin/`. Вход по `ADMIN_KEY` из `.env`. Показывает статистику (пользователи, заказы, выручка, активные подписки), таблицы заказов (с ручной выдачей ожидающих), подписок и пользователей. Пароли хранятся как scrypt-хеши; сессии — 90 дней.

> Без заданного `ADMIN_KEY` админ-эндпоинты и вход в панель заблокированы (это защита по умолчанию).

## API (кратко)

| Метод | Путь | Назначение |
|------|------|-----------|
| GET | `/api/plans` | Список тарифов |
| GET | `/api/config` | Конфиг для фронта (бренд, режимы, username бота) |
| GET | `/api/status` | Статус подключения к панели |
| POST | `/api/auth/register` · `/api/auth/login` | Регистрация / вход (возвращают токен сессии) |
| GET | `/api/auth/me` | Профиль и подписки (`Authorization: Bearer <token>`) |
| POST | `/api/auth/claim` | Привязать анонимные подписки к аккаунту |
| POST | `/api/checkout` | Создать заказ (`{planId}`); бесплатный тариф выдаётся сразу |
| POST | `/api/orders/:id/confirm` | Демо-подтверждение оплаты (если разрешено) |
| GET | `/api/subscription/:token` | Данные подписки: ключ, QR, срок, трафик |
| GET | `/api/admin/stats` · `/orders` · `/subscriptions` · `/users` | Данные для админки (заголовок `x-admin-key`) |

## Деплой

Это Node-приложение — разворачивайте на VPS (обычно там же, где 3x-ui). Держите процесс через `systemd`/`pm2`, а сайт проксируйте через Nginx с HTTPS. GitHub Pages не подойдёт (нужен backend).

Полная пошаговая инструкция для Ubuntu — в [**DEPLOY.md**](DEPLOY.md). Готовые артефакты в `deploy/`:
- `deploy/setup.sh` — установка одной командой (`sudo bash deploy/setup.sh`);
- `deploy/vibevpn.service` — systemd-юнит (автозапуск + рестарт);
- `deploy/nginx.conf.example` — reverse proxy для Nginx (+ certbot для HTTPS).
